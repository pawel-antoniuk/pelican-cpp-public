	PLC format specification

	Pawel Antoniuk

[INFO] VERSION:
	1.0

[LAYOUT<{ELEMENT /element}>(DEFAULT)] DEFAULT PLC LAYOUT:	- "/element" is a alias syntax
	[LAYOUT] PLC LAYOUT<{/element, RGB PIXEL}>

[LAYOUT<{ELEMENT, ?PIXEL}>] PLC LAYOUT:
	[SECTION] HEADER<{ELEMENT}>
	[VECTOR(|{ELEMENT}|)] ELEMENTS:							- "|{ELEMENT}|" is optional
		[SECTION] ELEMENT@DATA<?PIXEL>

[SECTION<{ELEMENT}>] HEADER:
	"PLC\0" [4B](str)										- magic word
	|{ELEMENT}| [4B](uint) NUMBER OF ELEMENTS				- the number of elements
	[VECTOR(|{ELEMENT}|)] METADATAS:						- metadata for each element in file
		[SECTION] ELEMENT@ID								- type of element (see "IDS OF ELEMENTS" CONSTANTS)
		[4B](uint)											- offset of data of the element relative to end of "HEADER" SECTION

[TYPE] ELEMENT:
	[SECTION] TEXTURE 2D:
		[SECTION] ID:
			1 [1B][uint]
		[SECTION<PIXEL>] DATA:
			[PIXEL TYPE: ID]								- type of pixel
			[4B](uint) WIDTH								- width of texture
			[4B](uint) HEIGHT								- height of texture
			[VECTOR(WIDTH)] ROWS:
				[VECTOR(HEIGHT)] COLUMNS: 
					[SECTION] PIXEL@DATA	

[TYPE] PIXEL:
	RGB PIXEL:
		[SECTION] ID:
			1 [1B](uint)
		[SECTION] DATA:
			[
				[1B](uint)	- red
				[1B](uint)	- green
				[1B](uint)	- blue
			]
	RGBA PIXEL:
		[SECTION] ID:
			2 [1B](uint)
		[SECTION] DATA:
			[
				[1B](uint)	- red
				[1B](uint)	- green
				[1B](uint)	- blue
				[1B](uint)	- alpha
			]
